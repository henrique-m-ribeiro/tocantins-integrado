# Metodologia de Pesquisa-Ação

## Visão Geral

Este projeto utiliza a metodologia de **pesquisa-ação** para documentar e analisar o processo de desenvolvimento e eventual incorporação de tecnologia de IA na gestão pública do estado do Tocantins.

## Definição

Segundo Tripp (2005), a pesquisa-ação é:

> "Uma forma de investigação-ação que utiliza técnicas de pesquisa consagradas para informar a ação que se decide tomar para melhorar a prática."

## Ciclos de Pesquisa-Ação

O projeto segue o modelo clássico de Kurt Lewin, com ciclos de:

```
    ┌─────────────┐
    │  PLANEJAR   │
    │  (Plan)     │
    └──────┬──────┘
           │
           ↓
    ┌─────────────┐
    │    AGIR     │
    │  (Action)   │
    └──────┬──────┘
           │
           ↓
    ┌─────────────┐
    │  OBSERVAR   │
    │ (Observe)   │
    └──────┬──────┘
           │
           ↓
    ┌─────────────┐
    │  REFLETIR   │
    │ (Reflect)   │
    └──────┬──────┘
           │
           └──────→ Próximo Ciclo
```

## Critérios de Qualidade

Seguimos os critérios de Heikkinen et al. (2007) para garantir a qualidade da pesquisa:

### 1. Continuidade Histórica
- Documentação da evolução do projeto ao longo do tempo
- Diários de pesquisa-ação de todas as sessões
- Registro de decisões e suas justificativas

### 2. Reflexividade
- Consciência do papel do pesquisador no processo
- Documentação de vieses e suposições
- Reflexões críticas em cada sessão

### 3. Dialética
- Inclusão de múltiplas perspectivas
- Documentação de tensões e contradições
- Registro de feedbacks de stakeholders

### 4. Trabalhabilidade
- Foco em resultados práticos e úteis
- Métricas de sucesso definidas
- Validação com usuários reais

### 5. Evocatividade
- Narrativa envolvente e acessível
- Uso de casos concretos e exemplos
- Documentação de momentos-chave
